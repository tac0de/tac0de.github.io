우리는 언제 그리고 어떻게 JavaScript memoization 을 사용해야 할까?

01 March, 2020 - 2 min read

Memoization 은 코드를 더욱 빠르게 작성할 수 있게 해주는 매우 흔히 사용되는 테크닉 입니다.

cache 를 이용해 결과를 저장하여, 후에 처리되는 '시간을 어느정도 잡아먹는 함수' 들이 같은 일을 나중에 반복 실행하지 않게 해줍니다.

이 정의를 통해 기준을 선정하여 언제 memoization 을 코드에 적용하면 될지를 도움 받을 수 있습니다.

  • Memoization 은 시간이 많이 소요되는, 성능이 느려지는 함수들의 속도를 개선하기 위해 중심적으로 사용되어야 합니다.
  • Memoization 은 후속 함수콜의 속도를 향상 시키기에, 특정 상황에서 여러개의 같은 함수 콜을 앞당기고 싶을 때 쓰입니다.
  • Memoization 은 결과를 메모리에 저장하기 때문에, 매우 다른 상황에서 여러개의 같은 함수 콜을 호출할 때는 피하는게 좋습니다.

객체 지향 자바스크립트 코드에서 memoization 을 쓰는 예는 다음과 같습니다.

class MyObject {
  constructor(data) {
    this.data = data
    this.data[this.data.length - 2] = { value: "Non-empty" }
  }

  firstNonEmptyItem() {
    return this.data.find(v => !!v.value)
  }

  firstNonEmptyItemMemo() {
    if (!this.firstNonEmpty) this.firstNonEmpty = this.data.find(v => !!v.value)
    return this.firstNonEmpty
  }
}

const myObject = new MyObject(Array(2000).fill({ value: null }))

for (let i = 0; i < 100; i++) myObject.firstNonEmptyItem() // ~4000ms
for (let i = 0; i < 100; i++) myObject.firstNonEmptyItemMemo() // ~70ms

위의 예제는 클래스 내에서 어떻게 memoization 을 적용시킬 수 있는지 보여주지만 객체의 라이프 사이클에서 data 구성이 변하지 않는다는 가정을 하게 됩니다.

그리고 이것이 유일하게 호출할 비싼(?) 함수 콜이기 때문에 재사용 할 수가 없습니다.

이는 또한 함수에 전달되는 인자에 대해서도 설명하지 않아 결과를 바꿀 수 있습니다.

주어진 함수와 함께 인자에 대해 설명하기 위한 기능적 접근방식은 다른 값을 저장하기 위해 Map 을 사용하는 memoize snippet 형태로 찾을 수 있습니다.

이 snippet 을 함수를 memoize 할때 주요하게 쓰라고 추천하지만, 자바스크립트의 Proxy object 는 handler.apply() trap 을 사용하여 흥미로운 대체법으로 제공될 수 있습니다.

다음과 같이 사용될 수 있습니다.

const memoize = fn =>
  new Proxy(fn, {
    cache: new Map(),
    apply(target, thisArg, argsList) {
      let cacheKey = argsList.toString()
      if (!this.cache.has(cacheKey))
        this.cache.set(cacheKey, target.apply(thisArg, argsList))
      return this.cache.get(cacheKey)
    },
  })

const fibonacci = n => (n <= 1 ? 1 : fibonacci(n - 1) + fibonacci(n - 2))
const memoizedFibonacci = memoize(fibonacci)

for (let i = 0; i < 100; i++) fibonacci(30) // ~5000ms
for (let i = 0; i < 100; i++) memoizedFibonacci(30) // ~50ms
