SEO (Search Engine Optimization) 은 검색 엔진 최적화로 인터넷이 커지면서 더욱 더 중요해지고 있습니다. 대부분의 웹 개발자들은 SEO 다루는 경험이 적거나 없을 수 있습니다. 그래서 기본적인 것에 대하여 딱히 어렵운 것도 아니니 4가지 팁을 소개하려 합니다.

### 1. 사용자 친화적인 URL 과 적절한 매핑

첫번째로 알아봐야 할 것은 URL slug 입니다. URL slug 는 사람이 읽기 편해야하며 '-' 로 단어들을 분리해야 합니다. 또한 무작위의 글자나 숫자를 추가해서도 안됩니다. URL의 부분들이 웹사이트의 논리적인 구조와 걸맞아야 합니다 (예. /blog/posts/seo-tips). 마지막으로 sitemap 을 만드시는데 시간을 조금 투자할 필요가 있습니다. 오래되거나 깨진 URL 을 새 URL 로 redirect 시켜주는게 404 페이지를 줄여주는데 도움을 줄 것입니다.

### 2. 구조가 잘 짜여진 데이터를 사용하여 구글이 웹페이지를 잘 이해할 수 있도록 도와주기

구글은 구조가 잘 짜여진 데이터를 통해 추천 알고리즘을 만드는데, 특정 검색을 하면 위에 카드 형태로 둘러싸있는 자료들을 볼 수 있는게 그 예입니다. 따라서 데이터를 잘 짬으로써, 검색결과에서 더 위에 뜰 것이고 심지어 맨 위에 구글이 자체적으로 추천하는 정보가 될 수도 있는 것입니다. 적절하게 구조화된 데이터 유형을 찾고 도입해 봅시다.

### 3. 구글 Analytics 와 구글 검색 콘솔 설정하기

새로운 팁이거나 그런건 아니지만, 너무나도 중요한 툴이므로 넣어봤습니다. 구글 Analytics 는 유저들의 행동과 문제점과 기회를 찾을 수 있게 해주는 데이터 수집을 가능하게 해줍니다. 그리고 구글 검색 콘솔은 유저들이 나의 웹페이지에 도달하기 전에 무엇을 검색하는지 알아내기에 유용합니다.

### 4. 마크업과 성능 그리고 접근성에 주의를 기울이기

이 팁은 아마 개발자라면 누구라도 염두해 두고 있을 것인 코드를 최적화 하는 일입니다. 시멘틱 마크업을 작성하고 요청을 최소화 하여 모든 기기 종류에 최적화 하여 웹사이트의 접근하기 편하고 빠르게 로딩되게 해줍시다. 웹사이트는 동적으로 움직이는 부분들이 많고 성장하고 시간이 지나면서 바뀌기 때문에 [Lighthouse](https://developers.google.com/web/tools/lighthouse) 같은 툴을 이용하여 주기적으로 나의 사이트를 심사 관리하는게 주는게 좋습니다.
